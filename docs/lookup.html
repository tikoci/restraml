      Enter a CLI path in CLI Path — results update automatically as you type (about 400 ms after you stop typing). Press Enter to search immediately. The last segment of the path determines what comes back:

      Version picks which RouterOS release to search. Toggle check all versions to scan every published release at once. include testing adds beta and RC builds to the version list.

      extra-packages searches schemas built with the full all_packages bundle, which adds: container, dude, iot, lora, openflow, tr069-client, user-manager, rose-storage, ups, wifi-qcom, wifi-qcom-ac, zerotier, calea, and gps (package list as of 7.22). Without this toggle, results reflect the base CHR image only.

      The Share button copies a direct link to your current search — version, path, attribute, and all toggle states are captured in the URL.

      Data comes from /console/inspect — the same introspection endpoint behind WinBox’s sidebar tree. A real RouterOS CHR boots in CI for each release, every API branch is walked via REST, and the result is published as inspect.json.  ·  README.md for tikoci/restraml
